# Security Policy

## Supported Versions

We adhere to Semantic Versioning. Only the latest major version receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

1.  **Do NOT create a public issue.**
2.  Email our security team at **arunkumarbind150720@gmail.com**.
3.  Include a detailed description of the vulnerability and steps to reproduce it.

## Response Timeline

- **Acknowledgement:** Within 24 hours.
- **Assessment:** Within 3 days.
- **Fix:** As soon as possible, depending on severity.

We request that you allow us reasonable time to fix the issue before public disclosure.

## Security Best Practices

We follow industry-standard security practices:

- Regular dependency updates.
- Automated security scanning in CI/CD.
- Secure coding guidelines (OWASP).
